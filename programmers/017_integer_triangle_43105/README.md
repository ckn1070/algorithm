# 정수 삼각형

## 📌 문제 설명
아래와 같은 삼각형 모양의 배열이 주어집니다.

맨 위층부터 시작하여 아래층으로 내려올 때,  
선택한 숫자에 대해 **바로 아래층의 왼쪽 또는 오른쪽 숫자**로만 이동할 수 있습니다.

이때, 거쳐간 숫자의 합이 최대가 되는 경로를 찾아  
그 **최댓값**을 반환하도록 `solution` 함수를 작성하세요.

---

## 📥 입력 조건
- `triangle`: 정수 삼각형이 담긴 2차원 배열
  - 삼각형의 높이: 1 이상 500 이하
  - `triangle[i]`의 길이 = `i + 1`
  - 각 원소: 0 이상 9,999 이하의 정수

---

## 📤 출력 조건
- 맨 위에서 맨 아래까지 이동하면서 얻을 수 있는 합의 최댓값을 반환

---

## 🔍 입출력 예시

| triangle | return |
|----------|--------|
| [[7],[3,8],[8,1,0],[2,7,4,4],[4,5,2,6,5]] | 30 |

---

## 🧠 해결 아이디어
- **동적 계획법(DP)**으로 각 위치까지의 최대 합을 누적합니다.
- `dp[i][j]`를 i번째 층 j번째 원소까지 내려왔을 때의 최대 합이라고 하면:
  - 왼쪽 끝(`j == 0`): `dp[i][j] = dp[i-1][0] + triangle[i][j]`
  - 오른쪽 끝(`j == i`): `dp[i][j] = dp[i-1][i-1] + triangle[i][j]`
  - 중간: `dp[i][j] = max(dp[i-1][j-1], dp[i-1][j]) + triangle[i][j]`
- 마지막 층 `dp[last]` 중 최댓값이 정답입니다.
- 메모리를 줄이려면 아래층 계산 시 1차원 배열로도 처리할 수 있습니다.

---

## 🧪 주의 사항
- 삼각형 높이가 최대 500이므로 **O(n^2)** DP는 충분히 가능합니다.
- 경로의 개수가 매우 많아 완전탐색은 비효율적입니다.
