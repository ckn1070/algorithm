# 더 맵게

## 📌 문제 설명
매운 것을 좋아하는 스코빌 지수가 낮은 음식들이 있습니다.  
모든 음식의 스코빌 지수를 **K 이상**으로 만들고 싶습니다.

음식을 섞는 방법은 다음과 같습니다.

- 가장 맵지 않은 음식의 스코빌 지수 +  
  (두 번째로 맵지 않은 음식의 스코빌 지수 × 2)

이렇게 만든 새로운 음식을 기존 음식 리스트에 다시 넣습니다.  
이 과정을 반복하여 모든 음식의 스코빌 지수가 K 이상이 되도록 하세요.

모든 음식의 스코빌 지수를 K 이상으로 만들기 위해  
**최소 몇 번 섞어야 하는지**를 반환하는 `solution` 함수를 작성하세요.

만약 모든 음식의 스코빌 지수를 K 이상으로 만들 수 없다면 **-1**을 반환합니다.

---

## 📥 입력 조건
- `scoville`: 음식의 스코빌 지수가 담긴 배열  
  - 길이: 2 이상 1,000,000 이하  
  - 각 원소: 0 이상 1,000,000 이하
- `K`: 원하는 스코빌 지수  
  - 0 이상 1,000,000,000 이하

---

## 📤 출력 조건
- 모든 음식의 스코빌 지수를 K 이상으로 만들기 위한 최소 횟수
- 불가능할 경우 `-1` 반환

---

## 🔍 입출력 예시

| scoville | K | return |
|--------|---|--------|
| [1,2,3,9,10,12] | 7 | 2 |

---

## 🧠 해결 아이디어
- 가장 맵지 않은 음식부터 빠르게 꺼내야 하므로 **우선순위 큐(최소 힙)**를 사용합니다.
- 힙에서 가장 작은 값 두 개를 꺼내 새로운 음식을 만듭니다.
- 새로 만든 음식을 다시 힙에 넣고 횟수를 증가시킵니다.
- 힙의 최솟값이 K 이상이 되면 종료합니다.

---

## 🧪 주의 사항
- 음식이 하나만 남았는데 K 미만이면 더 이상 섞을 수 없으므로 `-1`을 반환해야 합니다.
- 입력 크기가 매우 크므로 **정렬을 반복하는 방식은 비효율적**입니다.
- 시간 복잡도는 **O(n log n)** 수준으로 해결해야 합니다.
