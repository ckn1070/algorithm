# 프로세스

## 📌 문제 설명
운영체제의 역할 중 하나는 컴퓨터 시스템의 자원을 효율적으로 관리하는 것입니다.  
이 문제에서는 운영체제가 다음과 같은 방식으로 프로세스를 관리한다고 가정합니다.

1. 실행 대기 큐에서 프로세스를 하나 꺼냅니다.
2. 큐에 대기 중인 프로세스 중 **우선순위가 더 높은 프로세스가 하나라도 있으면**,  
   방금 꺼낸 프로세스를 다시 큐의 맨 뒤에 넣습니다.
3. 그렇지 않으면 해당 프로세스를 실행합니다.

각 프로세스의 우선순위를 나타내는 배열 `priorities`와  
궁금한 프로세스의 위치를 나타내는 `location`이 주어질 때,  
해당 프로세스가 **몇 번째로 실행되는지**를 반환하도록 `solution` 함수를 작성하세요.

---

## 📥 입력 조건
- `priorities`: 프로세스의 우선순위를 나타내는 배열  
  - 길이: 1 이상 100 이하
  - 각 원소: 1 이상 9 이하의 정수 (값이 클수록 우선순위가 높음)
- `location`: 알고 싶은 프로세스의 위치 (0-based index)

---

## 📤 출력 조건
- `location`에 해당하는 프로세스가 실행되는 순서를 반환  
  - 실행 순서는 **1부터 시작**

---

## 🔍 입출력 예시

| priorities | location | return |
|-----------|----------|--------|
| [2, 1, 3, 2] | 2 | 1 |
| [1, 1, 9, 1, 1, 1] | 0 | 5 |

---

## 🧠 해결 아이디어
- 프로세스를 `(우선순위, 원래 위치)` 형태로 **큐(Queue)**에 저장합니다.
- 큐에서 하나를 꺼낸 뒤:
  - 큐 안에 자신보다 우선순위가 높은 프로세스가 있는지 확인합니다.
  - 있다면 다시 큐의 뒤로 보냅니다.
  - 없다면 실행 순서를 증가시키고,
    해당 프로세스가 `location`이면 그 순서를 반환합니다.
- 실제 스케줄링 과정을 그대로 **시뮬레이션**하는 문제입니다.

---

## 🧪 주의 사항
- 실행 순서는 1부터 카운트합니다.
- 우선순위가 같은 프로세스는 **먼저 들어온 순서**를 유지합니다.
- 단순 정렬 문제로 착각하면 오답이 됩니다.
