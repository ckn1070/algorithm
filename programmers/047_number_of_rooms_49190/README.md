# 방의 개수

## 📌 문제 설명
원점(0,0)에서 시작해서 아래처럼 숫자가 적힌 방향으로 이동하며 선을 긋습니다.

스크린샷 2018-09-06 오후 4.55.33.png

예를 들어 1일 때는 오른쪽 위로 이동합니다.

그림을 그릴 때, 사방이 막히면 방 하나로 셉니다.  
이동하는 방향이 담긴 배열 `arrows`가 매개변수로 주어질 때, 방의 갯수를 `return` 하도록 `solution` 함수를 작성하세요.

---

## 📥 입력 조건
- 배열 `arrows`의 크기는 1 이상 100,000 이하입니다.
- `arrows`의 원소는 0 이상 7 이하입니다.
- 방은 다른 방으로 둘러 싸여질 수 있습니다.

---

## 📤 출력 조건
- 주어진 이동 경로로 선을 그렸을 때 만들어지는 **방의 개수**를 반환합니다.

---

## 🔍 입출력 예시

| arrows | return |
|---|---:|
| [6, 6, 6, 4, 4, 4, 2, 2, 2, 0, 0, 0, 1, 6, 5, 5, 3, 6, 0] | 3 |

---

## 🧠 해결 아이디어
- 이동 경로를 그래프로 보면, “방”은 새로운 **사이클(순환)**이 생길 때 증가합니다.
- 단, 대각선 이동으로 인해 “선이 교차”하는 지점이 방을 만들 수 있어,
  교차점을 정확히 정점으로 처리해야 합니다.
- 많이 쓰는 방법:
  - 좌표를 2배로 스케일 업(각 이동을 2번에 나눠서)하여 교차점을 격자 정점으로 만들기
  - 방문한 **정점(좌표)**, 방문한 **간선(두 정점 사이 이동)**을 set으로 관리
  - “이미 방문한 정점인데, 그 간선은 처음 지나가는 경우” → 새로운 방 1개 증가

---

## 🧪 주의 사항
- 같은 정점을 다시 방문해도, **이미 지나간 간선**으로 돌아오면 방이 새로 생기지 않습니다.
- 교차(대각선이 중간에서 만나는 경우)를 놓치면 방 개수가 틀어지므로,
  스케일 업(한 칸 이동을 2단계로 처리)이 사실상 핵심입니다.
- 간선은 방향이 있으므로 (A→B)와 (B→A)를 모두 기록해 “무방향 간선”처럼 관리하는 것이 안전합니다.
