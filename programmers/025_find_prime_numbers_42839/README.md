# 소수 찾기

## 📌 문제 설명
한 자리 숫자가 적힌 종이 조각이 흩어져 있습니다.  
이 종이 조각들을 붙여 **만들 수 있는 모든 수** 중에서  
**소수의 개수**를 구하려고 합니다.

종이 조각에 적힌 숫자를 문자열 `numbers`로 받았을 때,  
각 숫자를 **한 번씩만 사용하여** 만들 수 있는 모든 수 중  
소수가 몇 개인지 반환하도록 `solution` 함수를 작성하세요.

---

## 📥 입력 조건
- `numbers`: 숫자로만 이루어진 문자열  
  - 길이: 1 이상 7 이하
- 각 문자는 `'0'` ~ `'9'` 로 이루어져 있음

---

## 📤 출력 조건
- 만들 수 있는 수 중 **소수의 개수**를 반환

---

## 🔍 입출력 예시

| numbers | return |
|---------|--------|
| "17" | 3 |
| "011" | 2 |

---

## 🧠 해결 아이디어
- 가능한 모든 숫자 조합을 만들어야 하므로 **완전탐색**이 필요합니다.
- 숫자 조합 생성:
  - 문자열의 모든 길이(1 ~ n)에 대해 **순열(permutation)**을 생성합니다.
  - 중복된 숫자가 만들어질 수 있으므로 **set**을 사용해 중복 제거합니다.
- 생성된 각 숫자에 대해 **소수 판별**을 수행합니다.
  - 2부터 √n까지 나누어 떨어지는지 확인합니다.
- 소수인 경우만 카운트합니다.

---

## 🧪 주의 사항
- `"011"` → `11`, `011`, `101` 처럼  
  앞자리가 0인 경우도 숫자로 변환 시 정상 처리해야 합니다.
- 0과 1은 소수가 아니므로 반드시 제외해야 합니다.
- 최대 길이가 7이므로 최악의 경우의 수(7! + …)도 충분히 계산 가능합니다.
