# 가장 먼 노드

## 📌 문제 설명
n개의 노드가 있는 그래프가 있습니다.  
각 노드는 1번부터 n번까지 번호가 매겨져 있으며, **모든 간선은 양방향**입니다.

1번 노드에서 가장 멀리 떨어진 노드가 몇 개인지 구하려고 합니다.  
여기서 거리란 **간선의 개수**를 의미합니다.

1번 노드로부터 **최단 경로로 이동했을 때 가장 먼 노드들의 개수**를 반환하도록  
`solution` 함수를 작성하세요.

---

## 📥 입력 조건
- `n`: 노드의 개수  
  - 2 이상 20,000 이하
- `edge`: 간선 정보가 담긴 2차원 배열  
  - 길이: 1 이상 50,000 이하
  - 각 원소는 `[a, b]` 형태이며, a번 노드와 b번 노드가 연결되어 있음
- 모든 간선은 **양방향**
- 그래프는 항상 연결되어 있음

---

## 📤 출력 조건
- 1번 노드에서 가장 멀리 떨어진 노드의 개수를 반환

---

## 🔍 입출력 예시

| n | edge | return |
|---|------|--------|
| 6 | [[3,6],[4,3],[3,2],[1,3],[1,2],[2,4],[5,2]] | 3 |

---

## 🧠 해결 아이디어
- 가중치가 없는 그래프이므로 **BFS(너비 우선 탐색)**를 사용해  
  1번 노드로부터 각 노드까지의 최단 거리를 구합니다.
- BFS 과정에서 각 노드의 거리를 배열에 저장합니다.
- 모든 노드의 거리 중 **최댓값**을 구한 뒤,
  해당 거리와 같은 값을 가진 노드의 개수를 셉니다.

---

## 🧪 주의 사항
- 노드 수와 간선 수가 많으므로 **인접 리스트**로 그래프를 구성해야 합니다.
- 방문 여부와 거리를 동시에 관리하면 효율적으로 탐색할 수 있습니다.
