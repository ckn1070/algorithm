# 여행경로

## 📌 문제 설명
주어진 항공권을 모두 이용하여 여행경로를 짜려고 합니다. 항상 "ICN" 공항에서 출발합니다.

항공권 정보가 담긴 2차원 배열 `tickets`가 매개변수로 주어질 때, 방문하는 공항 경로를 배열에 담아 `return` 하도록 `solution` 함수를 작성해주세요.

---

## 📥 입력 조건
- 모든 공항은 알파벳 대문자 3글자로 이루어집니다.
- 주어진 공항 수는 3개 이상 10,000개 이하입니다.
- `tickets`의 각 행 `[a, b]`는 `a` 공항에서 `b` 공항으로 가는 항공권이 있다는 의미입니다.
- 주어진 항공권은 모두 사용해야 합니다.
- 만일 가능한 경로가 2개 이상일 경우 알파벳 순서가 앞서는 경로를 `return` 합니다.
- 모든 도시를 방문할 수 없는 경우는 주어지지 않습니다.

---

## 📤 출력 조건
- 모든 항공권을 정확히 한 번씩 사용하여 방문하는 **공항 경로**를 배열로 반환합니다.
- 가능한 경로가 여러 개라면 **알파벳 순으로 가장 앞서는 경로**를 반환합니다.

---

## 🔍 입출력 예시

| tickets | return |
|---|---|
| [["ICN", "JFK"], ["HND", "IAD"], ["JFK", "HND"]] | ["ICN", "JFK", "HND", "IAD"] |
| [["ICN", "SFO"], ["ICN", "ATL"], ["SFO", "ATL"], ["ATL", "ICN"], ["ATL","SFO"]] | ["ICN", "ATL", "ICN", "SFO", "ATL", "SFO"] |

---

## 🧠 해결 아이디어
- 모든 항공권을 “한 번씩” 사용해야 하므로, 간선을 모두 사용하는 경로를 찾는 문제로 볼 수 있습니다.
- 출발지는 항상 "ICN"으로 고정입니다.
- 가능한 경로가 여러 개인 경우 “알파벳 순으로 가장 앞서는 경로”를 반환해야 하므로,
  다음 목적지를 선택할 때 사전순 기준을 만족하도록 탐색 순서를 정해야 합니다.
- 대표적으로 DFS(백트래킹)로 경로를 구성하거나,
  그래프에서 모든 간선을 사용하는 경로(오일러 경로) 관점으로 스택을 활용해 해결할 수 있습니다.

---

## 🧪 주의 사항
- 항공권은 중복될 수 있으므로(동일한 [a,b]가 여러 장) “사용 여부”를 정확히 관리해야 합니다.
- 경로가 여러 개일 수 있으니, 탐색 시 사전순 조건을 항상 만족하도록 정렬/우선순위 처리가 필요합니다.
