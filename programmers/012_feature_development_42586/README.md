# 기능개발

## 📌 문제 설명
프로그래머스 팀에서는 기능 개선 작업을 수행 중입니다.  
각 기능은 진도가 100%일 때 서비스에 반영할 수 있습니다.

각 기능의 개발 속도는 모두 다르기 때문에,  
뒤에 있는 기능이 먼저 개발되더라도 **앞에 있는 기능이 배포될 때 함께 배포**됩니다.

먼저 배포되어야 하는 순서대로 작업의 진도가 담긴 배열 `progresses`와  
각 작업의 개발 속도가 담긴 배열 `speeds`가 주어질 때,  
각 배포마다 몇 개의 기능이 배포되는지를 배열로 반환하도록 `solution` 함수를 작성하세요.

---

## 📥 입력 조건
- `progresses`: 작업의 진도가 담긴 배열  
  - 길이: 1 이상 100 이하
  - 각 원소: 0 이상 100 미만
- `speeds`: 작업의 개발 속도가 담긴 배열  
  - 길이: `progresses`와 동일
  - 각 원소: 1 이상 100 이하

---

## 📤 출력 조건
- 각 배포마다 배포되는 기능의 개수를 담은 배열을 반환

---

## 🔍 입출력 예시

| progresses | speeds | return |
|-----------|--------|--------|
| [93, 30, 55] | [1, 30, 5] | [2, 1] |
| [95, 90, 99, 99, 80, 99] | [1, 1, 1, 1, 1, 1] | [1, 3, 2] |

---

## 🧠 해결 아이디어
- 각 기능이 완료되기까지 필요한 날짜를 계산합니다.
  - `남은 작업량 = 100 - progress`
  - `필요 일수 = ceil(남은 작업량 / speed)`
- 앞 기능의 완료일을 기준으로 뒤 기능들을 확인하며 **같은 배포 묶음**을 만듭니다.
  - 현재 배포 기준일보다 완료일이 작거나 같으면 함께 배포
  - 더 크면 새로운 배포로 시작
- 큐(또는 인덱스)로 순서대로 처리하면 됩니다.

---

## 🧪 주의 사항
- 배포는 반드시 **앞에서부터 순서대로** 이루어집니다.
- `ceil` 처리를 정확히 해야 합니다. (정수 나눗셈 시 올림 처리)
- 입력 크기는 크지 않지만, 구현 실수가 잦은 유형입니다.
